### Binary Data Format

Confirmed the vendor record structure:

- There is no global file header.
- Files are repeated records.
- Each record has an 18-byte header:
  - `seq`: 4 bytes, little-endian unsigned int
  - `data_len`: 4 bytes, little-endian unsigned int
  - `timestamp`: 10 bytes
- Timestamp layout:
  - year offset from 2000
  - month, day, hour, minute, second
  - microsecond as 4-byte big-endian unsigned int
- Content layout:
  - 256 sample groups
  - 7 channels per group
  - 3 bytes per channel
  - channel order: `Ua, Ub, Uc, Ia, Ib, Ic, I0`
- Current parser decodes channel samples as signed little-endian int24.

### Feature Extraction

- Implemented `scripts/extract_features.py`.
- Generated minute-level features from raw waveform files.
- Extracted features include:
  - per-channel RMS, mean, std, min, max, peak-to-peak, crest factor
  - harmonic-like features: `thd_like`, 3rd/5th/7th harmonic ratios
  - current and voltage unbalance
  - zero-sequence current ratio
  - power-like relative features: `P_total`, `S_total`, `power_factor_like`
- Current values are still raw int24-derived features, not calibrated physical
  units.

### Harmonic and Interference Check

- Voltage waveform distortion is relatively low:
  - voltage `thd_like` median around 3.7%
  - 99th percentile around 5.7%
- Current waveform distortion is much stronger:
  - current `thd_like` median around 39%-45%
- Current harmonics are not treated as pure interference because they may be
  useful load signatures.
- Current plan is to keep harmonic features and later add fundamental-only
  features for comparison.

## Current Limitations

- There are no real equipment operation labels yet.
- Existing labels are pseudo labels from waveform feature rules.
- Stop events are severely underrepresented.
- Voltage/current scaling factors are not confirmed, so features are not yet in
  physical units.
- The current model identifies event type, not specific equipment identity.

## Recommended Next Steps

1. Obtain a small set of real labels, especially stop events.
2. Ask for equipment operation logs, PLC records, test switching periods, or
   confirmed production start/stop times.
3. Build a dedicated down-event detector to separate stop, load reduction, and
   state-change cases.
4. Confirm voltage/current scaling factors.
5. Add fundamental-only features while preserving harmonic features.
6. Retrain after real labels or better stop-event pseudo labels are available.
